    Service in Angular - Why Service?
    Dont Repeat same code frequently in various components
    Write once use it various components (providing data for various components) thats SERVICE

    How to create a Service?
    1. In Terminal ng g s servicename
    2. Put the service name in App.Module.ts file like providers: [servicename] and also import it on top of the file.
    3. Then import service in ts file (our required component) and also declare it like constructor(private _sampleservice:SampleService) same file.

    Error Handling

             * If we get a error from url response we have to display it what kind of error comes so we use Error Handling

    How?

    1.Open Service.ts file and import import catchError from 'rxjs/operators'; and also import import throwError as observableThrowError, Observable from 'rxjs';

    2.Add this return this.http.get student[] (this._errorurl).pipe( catchError(this.errorHandler));

    3.Error handler function look like this errorHandler(error: HttpErrorResponse) return observableThrowError(error.message || "Server Error");

    4.Then get error in to our TS file by like .subscribe(data=>this.studentdata = data, error => this.errormsg = error); under the service integrated function

    5.Now define globally Error Msg in TS file for display the error in our HTML file. like public errormsg;.

    6.Finally display error msg as interpolation.

    Routing in Angular

    Basic steps of Routing

    1.First create a new routing option enabled project by execute cmd ng new projectname --routing by creating like this it will automatically created App.routing.ts file itself.

    2.In other case we have existing project without routing option follow these steps.

          * Open index.html and ensure there is a base href="/" tag at the top of the head section.

          * In the root directory of the application, run the command: ng g module app-routing.

    3.Now open App.routing.ts and import RouterModule, Routes from '@angular/router';

    4.Then Declare it imports: [ CommonModule, RouterModule.forRoot(routes) ], exports: [ RouterModule ],

    5.Then Create Components what we need and import it.

    6.Now Define the Path in App.routing.ts like const routes: Routes = [ path:'dashboard',component: DashboardComponent, path:'users',component:UsersComponent ];

    7.Then add export const routingComponents = [DashboardComponent,UsersComponent] in App.routing.ts because we declare all routing components here we dont need to import same in App.module.ts like this import AppRoutingModule, routingComponents from './app-routing/app-routing.module';

    8.Dont forget to add router-outlet /router-outlet tag in html file our component will render in that tag.

    Wildcard Routing

    1.If we type wrong url path follwed by '/' we will show a message Page Not Found so we use it.

    2.By adding this path:'**',component:PageNotFoundComponent in App.routing.ts file. It declare at the end of the components declarations.Dont add it earlier beacause it works all the times when page is loaded.

    3.Now we do another one thing if our url like http://localhost:4200 without any specific path we define if empty url comes what page will render so, add path:'',redirectTo:'/dashboard',pathMatch:'full' this empty url path comes it redirectTo what component we defined.pathMatch have a two property like prefix , full full means full path is empty.

    Passing Parameter in URL

    1.Create a list (in here is the example) and write a (Click) function on the iterated lists and passing parameter like (click)="onSelect(getpage)" getpage denoted *ngFor="let getpage of rdata"

    2.Add codes in the function like onSelect(getpage) this.router.navigate(['/dashboard',getpage.id]); let id = parseInt(this.route.snapshot.paramMap.get('id')); this.pageid = id; and also import import ActivatedRoute from '@angular/router' in ou corresponding TS.file

    3.snapshot.paramMap is used for get data from URL.

    4.Finally add path:'dashboard/:id',component:CompanydetailsComponent in App.routing.ts

    Params In Angular

    1. Add the function Buttons like Prev and Next

    2.Let write in the function button (Prev Button) let prevbtn = this.pageid - 1; this.router.navigate(['/dashboard',prevbtn]);

    3.Then write in the function button (Next Button) let nxtbtn = this.pageid + 1; this.router.navigate(['/dashboard',nxtbtn]);

    4.Now write code in ngOnInit() in same TS.file

    5.On the ngOnInit put the code like this.route.paramMap.subscribe((params:ParamMap) => let id = parseInt(params.get('id')); this.pageid = id;

    6.Params is used for overcome the snapshot probelms

    7.On url will change based on ID but in the component view not possible for change so overcome this we use Parms.

    Relative Navigation

    1.If we change our component name in App.routing.ts after some time we want change in all components which is changed in app.routing.ts so overcome this so we use Relative Navigation

    2.Like this this.router.navigate([getpage.id],relativeTo: this.route);

    Angular 6 ?


    1.Angular 6 comes with all branch upadtions like Angular framework , CLI, Angular Material

    2.We use animations we import from @angular/core in earlier version but now import from @angular/animations

    3.ng-template tag changed now template

    4.Before that we register service in providers section now service file contains a buit in option providedIn:'root' now no need to add service in provider section in App.module.ts

    5.In Angular 6 we can easily modify changes by ng update which is update our package.json file and providing updates for our application.

    6.Another one is ng add if we add a new module in existing project by run yhis command.

    7.Finally Ivy is used for increase speed of application and reduced the file size but it not integrated now this on stand-by.
